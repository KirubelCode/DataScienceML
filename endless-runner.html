Learn More - Endless Runner Game

The Endless Runner Game is a dynamic and engaging project inspired by the mechanics of Geometry Dash. This game was built using x68 assembly language, showcasing a deep understanding of low-level programming concepts.

Acknowledgment of Template

This project is built upon a starter template provided by [Philip Bourke] for course [SE603]. The template offered a foundational structure, including trap codes for screen size, key inputs, and double buffering. Modifications were made to expand functionality and transform the basic structure into a complete game experience.

Modifications Implemented

Below are the key features and modifications added to create the Endless Runner Game:

  • Text Display: Added in-game text updates for difficulty levels, player score, and lives remaining. Displays game over messages and prompts for restart.
  • Sound Integration: Incorporated sound effects for player actions (jumping, collisions) and game events.
  • Player Mechanics: Developed player movement mechanics, including constant X-axis movement, jumping, and landing.
  • Game Progression: Designed a dynamic difficulty system where enemy speed increases based on player score.
  • Game Over and Restart: Implemented a system where losing 3 lives ends the game, with an option to restart by pressing Enter.

Gameplay Overview

The player controls a character that constantly moves along the X-axis while dodging enemy blocks. The challenge lies in jumping over enemies, with the difficulty increasing as the game progresses. Colliding with an enemy results in a loss of one life, and losing all three lives ends the game. The player can restart the game by pressing Enter.

Technical Details

The Endless Runner Game uses x68 assembly language to directly manage system-level operations, including graphics rendering, input handling, and game logic. Trap codes are utilised for:

  • Screen size information and double buffering.
  • Listening for key inputs (e.g., spacebar for jumping, escape for exiting).
  • Setting cursor positions and displaying text.

Additionally, custom algorithms were implemented for player movement, gravity simulation, enemy speed adjustment, and collision detection.
